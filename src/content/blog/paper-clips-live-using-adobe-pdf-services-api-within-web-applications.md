In this episode of Paper Clips on September 29th, 2021 at 9:00AM PST, our live stream Ask the Experts session for Adobe Document Services, we will cover some of how you can use Adobe PDF Services API within your web apps.

You can subscribe to Paper Clips by subscribing to Adobe Developers on YouTube or Adobe Document Cloud on LinkedIn. We will also answer questions from the audience about anything about Adobe Document Services.


## News Highlights

### NEW: PDF Properties API

![](/blog/paper-clips-live-using-adobe-pdf-services-api-within-web-applications/02.png)

In the September release of Adobe PDF Services API, you can now get information about a PDF that can help decide how to use other PDF Services APIs such as OCR, Protect, Split, Export, and other services.

For more information, see:

- Release notes
- Unpacking the September Release of Adobe Document Services

### UPDATED: HTML to PDF

![](/blog/paper-clips-live-using-adobe-pdf-services-api-within-web-applications/03.png)

Before the September release of Adobe PDF Services API, you have been able to convert HTML to PDF by uploading a ZIP file to Adobe PDF Services API to generate a PDF. You can now pass HTML directly to the service as HTML with inline CSS or provide a URL to convert to PDF in the new release.

For more information, see:

- Release notes
- Unpacking the September Release of Adobe Document Services

### UPDATED: .NET SDK

If you’re a .NET developer, support has now been added to the Adobe PDF Services SDK for .NET. Follow the normal instructions for including the SDK and you’ll have Extract support for your application.
